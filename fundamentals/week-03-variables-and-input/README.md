# Week 03 — Lists and Console Input

This week you learn your first **data structure**: the Python list.
You will also learn how to interact with your programs through the terminal using `input()`.

---

## What you will learn

- What a **data structure** is and why grouping data matters
- How to create **lists** of strings, numbers, mixed types, and lists of lists
- **Index positions** — 0-based indexing, negative indexing with `-1`, and why it works this way
- Combining **f-strings with index access**: `f"{my_list[2]}"`
- **Modifying** elements in a list by index
- What a **method** is and common list methods: `len()`, `.append()`, `.insert()`, `.remove()`
- **Slicing** lists with `[start:end]`
- Why **strings are sequences** and can be indexed and sliced just like lists
- **`input()`** — pausing a program to collect text from the user

---

## How to run the materials

**Option 1 — Marimo VS Code extension (easiest)**
Open this file in VS Code:

```
fundamentals/week-03-variables-and-input/notebooks/nb_lists_and_input.py
```

**Option 2 — Terminal (Marimo in the browser)**

```sh
marimo run fundamentals/week-03-variables-and-input/notebooks/nb_lists_and_input.py
```

**Option 3 — Plain script (press F5 in VS Code)**

```sh
python fundamentals/week-03-variables-and-input/scripts/lists_and_input.py
```

> The script uses `input()` — run it in the terminal to experience typing responses live.
